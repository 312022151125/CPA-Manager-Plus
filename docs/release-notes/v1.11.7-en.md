## Overview

This release adds resumable usage-history import and permanent hourly aggregates while retaining raw events and safe fallback paths for large-scale data import and analytics. Credential inspection now presents more accurate action outcomes and logs, verified paid xAI OAuth credentials use the official API for real-inference checks, and editing drawers no longer close accidentally during text selection.

## Highlights

### Features

- Support session-based chunk upload, resume, cancellation, and visible progress for usage-history imports beyond the legacy single-request limit (`manager-server/usage`, `web/monitoring`).
- Serve eligible Dashboard and Usage Analytics queries from permanent UTC hourly aggregates together with recent raw data. Unsupported filters and timezone boundaries safely fall back to raw-event queries (`manager-server/usageaggregate`).
- Load Monitoring analytics by the active tab and separate account/API-key selectors from metric requests, avoiding unnecessary work for inactive views (`web/monitoring`).
- Preserve detailed automated-action outcomes, execution logs, and manual-review conflicts in credential inspection while keeping browser-persisted details sanitized (`web/monitoring`, `manager-server/codexinspection`).

### Fixes

- Route real-inference checks for paid xAI OAuth credentials with verified official identity through the official Responses API; the existing CLI route remains the default when that evidence is absent (`web/quota`, `manager-server/codexinspection`).
- Make usage-timeline cost and weighted-latency totals independent of storage row order, and keep identity first-seen timestamps stable across legacy duplicate imports (`manager-server/analytics`).
- Close drawers only when the same pointer starts and ends on the overlay, preventing accidental closure while drag-selecting input text. Intentional overlay click-to-close behavior is unchanged (`web/components/ui`).

## Upgrade Notes

- Manager Server automatically creates the SQLite tables required for import sessions and usage aggregates. Raw usage events are retained, and this phase does not enable historical-data deletion.
- The legacy `/import` single-request endpoint remains limited to 64 MiB. Use the new Manager Server panel session-import flow for larger history files.
- Import sessions default to 4 MiB chunks, a 16 GiB temporary disk quota, two concurrent sessions, and a 24-hour retention period. Administrators can adjust the related configuration for their deployment capacity. Resume keys can recover their corresponding sessions and must remain private.
- The xAI routing fix requires no configuration change. Only paid OAuth credentials with verified official identity switch to the official API.
