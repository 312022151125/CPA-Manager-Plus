## Overview

本次发布为用量数据引入可续传导入与持久化小时汇总，在保留原始事件和兼容回退路径的前提下，让大规模历史数据导入和监控分析更可靠。同时，凭证巡检会呈现更准确的执行结果与日志，已验证的付费 xAI OAuth 凭证将使用官方 API 完成真实推理检查，并修复编辑抽屉在拖选文本时意外关闭的问题。

## Highlights

### Features

- 用量历史支持基于会话的分块上传、续传、取消与进度展示，可处理超过旧版单请求限制的大文件（`manager-server/usage`、`web/monitoring`）。
- Dashboard 和用量分析可从持久化 UTC 小时汇总与最新原始数据共同读取；不支持的筛选条件和时区边界会安全回退到原始事件查询（`manager-server/usageaggregate`）。
- 监控中心按当前标签加载分析数据，并将账户和 API Key 选择器与指标请求分离，减少无关标签的查询工作（`web/monitoring`）。
- 凭证巡检保留自动操作的详细结果、执行日志和待人工处理的冲突，同时继续对浏览器持久化内容进行脱敏（`web/monitoring`、`manager-server/codexinspection`）。

### Fixes

- 已验证官方身份的付费 xAI OAuth 凭证会通过官方 Responses API 执行真实推理检查；缺少该证据时仍保留既有 CLI 路由（`web/quota`、`manager-server/codexinspection`）。
- 用量时间线的费用与加权延迟统计不再依赖存储行顺序，历史重复导入的身份首次发现时间也保持稳定（`manager-server/analytics`）。
- 抽屉仅在同一指针于遮罩按下并释放时关闭，避免在编辑输入框中拖选文本时意外关闭；直接点击遮罩关闭的行为保持不变（`web/components/ui`）。

## Upgrade Notes

- Manager Server 会自动创建导入会话和用量汇总所需的 SQLite 表；原始用量事件会保留，本阶段不会启用历史数据删除。
- 旧版 `/import` 单请求接口仍限制为 64 MiB。更大的历史文件应通过 Manager Server 面板中的新会话导入流程处理。
- 导入会话默认使用 4 MiB 分块、16 GiB 临时磁盘配额、最多 2 个并发会话和 24 小时保留期；管理员可按部署容量调整相关配置。恢复密钥可恢复对应会话，应仅由管理员保管。
- xAI 路由修复不需要额外配置；只有已验证官方身份的付费 OAuth 凭证会切换到官方 API。
