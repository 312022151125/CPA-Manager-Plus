# Grok 账号巡检

Grok 账号巡检通过 **账单** 周/月额度检查 xAI/Grok 认证文件（cli-chat-proxy 接口）。这是 **仅本地** 功能：在当前浏览器会话中运行，上次结果写入 localStorage，不会在 Manager Server 上调度。

免费额度耗尽（`subscription:free-usage-exhausted`）请走配置里的 **Grok/xAI 免费额度冷却**。那条路径与本页无关。

## 检查什么

- 来自 `fetchXaiQuota` 的周/月账单窗口。
- 用量是否达到禁用阈值（默认 100% = 完全耗尽）。
- 已禁用文件是否恢复，可建议启用。
- 账单返回 401/403 时建议到 [OAuth 登录](./oauth.md) 重新授权。
- 探测失败保持 **保留** 并附错误详情，不会因账单失败自动删除。

## 建议动作

- **保留**：健康或已处理。
- **禁用**：用量 ≥ 阈值且文件仍启用。
- **启用**：文件已禁用且用量低于阈值。
- **重新登录**：账单鉴权失败；去 OAuth 后重新巡检。
- **删除**：仅在你对需重新登录账号选择删除时（需确认）。

动作经现有认证文件 API 在确认后执行。

## 设置

仅本机浏览器设置：

- 额度阈值（%）
- 抽样数量（`0` = 全部 xAI 文件）
- 探测并发 / 重试 / 超时元数据
- 自动动作模式（无 / 启用 / 禁用 / 删除）
